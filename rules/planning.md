# Planning Principles

## Core Principles
- **Document once, reference many** — PROJECT.md as single source of truth
- **Timestamp everything** — chronological clarity
- **Include context** — why matters as much as what
- **Define success** — clear completion criteria

## Stay in Planning When
- Root cause unclear
- Multiple approaches possible
- Risk assessment incomplete
- Dependencies unverified

## Move to Implementation When
- Problem understood
- Approach validated
- Risks assessed
- Dependencies confirmed

## Related Commands
- `/create-feature` — End-to-end feature and planned refactor workflow
- `/fix-bug` — Bug workflow that invokes internal planning only when needed
- `/start` — Initialize work session
- `/update-project-file` — Refresh PROJECT.md
