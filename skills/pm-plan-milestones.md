# Plan Milestones

Use this phase when the feature needs milestone framing beyond a single small implementation slice.

## Goal

Break the work into product-level milestones that make scope and delivery sequencing clear before technical PR planning starts.

## Core Steps

1. Decide whether milestone framing is needed or whether one slice is enough.
2. Define the smallest meaningful milestone sequence.
3. Keep milestones product-facing rather than code-layer-specific.
4. Make sure each milestone has a clear success condition.
5. Each success condition should be verifiable — "users can X" not "X is implemented." Success conditions become the acceptance criteria that tell the team when the milestone is done.

## Output

```markdown
## Milestones

- Milestone: <name>
  - Outcome: <what is delivered>
  - Success condition: <how we know it is complete>
```
