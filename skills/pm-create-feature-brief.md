# Create Feature Brief

Use this phase when a workflow needs a clear product-facing brief before technical planning.

## Goal

Turn a raw feature request into a brief that is specific enough for technical planning and later validation.

## Core Steps

1. Restate the request in user-facing terms.
2. Define goals, non-goals, and assumptions.
3. Write clear acceptance criteria.
4. Note dependencies, rollout considerations, and important risks.
5. Keep the brief compact and decision-oriented.

## Output

```markdown
## Feature Brief

- Goal: <what changes for the user or system>
- Non-goals: <what this does not include>
- Acceptance criteria:
  - <criterion>
- Assumptions:
  - <assumption>
- Risks:
  - <risk>
```
